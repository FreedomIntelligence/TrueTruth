关于"流式输出"的说明
后端已支持的能力：
· LLM 已有 stream_reasoning() 方法（流式 token）
· SSE 长连接已在运行（每个 stage 事件）
· stdout 捕获 → AGENT_LOG 事件
需要新增的：
· 新 SSE 事件类型 AGENT_TOKEN
· Apply agent 调用流式 LLM 并 forward tokens
· 前端接收 token → 渐进显示
