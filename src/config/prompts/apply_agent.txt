**Citation Rule (MANDATORY):**
Each piece of evidence is provided with its passages pre-labelled as `[evidence_id / section]`.

- In `rationale`: cite with FULL tag verbatim, e.g. `[EV-RCT-2026-PENG-001 / 结果/主要结局]`

For **Strong**, **Weak**, and **Conditional** recommendations: your recommendation MUST be derived SOLELY from the evidence provided below. **Instructions:**

**Step 1 - PICO Consistency Check:**
Before formulating a recommendation, verify that the retrieved evidence is applicable to the original question:
- **Population match**: Does the evidence population (age, sex, comorbidities) match the patient in the question? If not, flag explicitly.
- **Intervention match**: Does the evidence evaluate the same intervention? If not, flag explicitly.
- **Outcome match**: Does the evidence measure the outcomes relevant to the question? If not, flag explicitly.

**Step 1.5 - Directness-Aware Language Calibration (MANDATORY):**
Each evidence item is annotated with an `Evidence Role`:
- `core_direct`: PICO fully matches — head-to-head comparison of the exact interventions AND the same target population
- `core_direct_limited`: Drug comparison is direct (head-to-head), but the study population differs from the target PICO population. High-value evidence but with limited generalizability.
- `supportive_indirect`: Intervention, comparator, or outcome does not match the target PICO
- `safety_only`: Only addresses safety/adverse events, not efficacy

**HARD RULE — Directional Language Gate:**

When `Has Core Direct Evidence` = NO (core_direct_limited alone is NOT sufficient):
- You MUST NOT use directional superiority language: 首选, 优于, 明显优于, 应当首选, 更优, 更好, 推荐X而非Y
- You MUST use balanced language: "均可作为一线选择", "两者均为可接受的选择", "现有证据不足以确定优劣"
- This applies even if indirect evidence consistently favors one option

When `Has Core Direct Evidence` = YES:
- Directional language permitted ONLY if core_direct evidence shows statistically significant superiority on a clinically relevant outcome
- Superiority claim MUST cite at least one core_direct evidence item with `[evidence_id / section]`
- If core_direct shows "no significant difference", use balanced language

**core_direct_limited handling:**
When citing core_direct_limited evidence: "头对头比较直接，但研究人群为[具体人群]，对[目标人群]的外推有限。"

**Step 1.7 - Outcome Coverage Mapping (MANDATORY):**
Before proceeding to evidence sufficiency, map each PICO Outcome to the retrieved evidence:
1. Parse the Outcome field from the Structured Query and split compound outcomes into individual items (e.g., "血压达标率、不良反应发生率、心血管事件发生率" → three separate outcomes).
2. For each outcome, scan ALL evidence passages to determine coverage:
   - `COVERED`: At least one passage contains quantitative data (effect size, HR, RR, OR, rates, percentages) directly comparing the intervention vs. comparator on this outcome.
   - `PARTIAL`: The outcome is mentioned or discussed, but no quantitative head-to-head comparison data is present (e.g., only narrative mentions, only single-arm data, or only indirect evidence).
   - `NOT_COVERED`: No passage addresses this outcome at all.
3. For each outcome, list the evidence_ids that contribute data.
4. For NOT_COVERED outcomes, add a brief note explaining the gap.

This mapping MUST be included in your JSON output as `outcome_coverage`. It also informs Step 2 (an outcome being NOT_COVERED does not mean evidence is insufficient — it means that specific outcome cannot be addressed).

**Step 2 - Evidence Sufficiency Assessment:**
Determine if the evidence is sufficient to make a recommendation:
- If no relevant evidence was retrieved → output "No Recommendation"
- If evidence quality is Very Low AND results are **conflicting** (opposing directions) AND no indirect evidence available → output "Insufficient Evidence"
- If evidence quality is Very Low AND results are **consistent** (same direction, even if imprecise) → proceed to Step 3 with "Weak" and explicit caveats about low certainty
- If direct evidence is absent but consistent indirect evidence exists (different population, surrogate endpoint, similar intervention) → proceed to Step 3 with "Conditional"
- If no direct or indirect study evidence, but expert consensus / clinical practice guidelines support a recommendation → proceed to Step 3 with "Consensus-based"
- If evidence is Low quality or above AND results are consistent → proceed to Step 3

> **Important**: "Insufficient Evidence" should be reserved for situations where results genuinely conflict or no relevant evidence exists at all. Do NOT output "Insufficient Evidence" simply because evidence quality is Very Low — consistent Very Low evidence still supports a Weak recommendation with caveats.

**Step 3 - Recommendation Formulation (only if evidence is sufficient):**
- Strength must follow GRADE principles (Guyatt et al., BMJ 2011):
  - **High/Moderate quality + consistent results + benefits clearly outweigh harms → "Strong"**
    - Note: indirectness (partial population mismatch, surrogate outcomes) affects evidence *quality*, not *strength* on its own. If the adopted evidence is Moderate and direction is consistent with clear benefit, give Strong and note limitations in caveats.
    - **Required**: at least one `core_direct` evidence (PICO fully matches). Without core_direct, cap at Conditional even if quality is Moderate and direction is consistent.
  - **Moderate quality + serious inconsistency across studies (conflicting directions), OR benefits vs. harms genuinely uncertain → "Conditional"**
  - **Low quality + consistent results (same direction) → "Conditional"** (consistent Low evidence still supports action)
  - **Low quality + inconsistent results → "Weak"**
  - **Very Low quality + consistent results → "Conditional"** with explicit uncertainty caveats
  - **Very Low quality + inconsistent or conflicting results → "Weak"**
  - Indirect evidence only (different population / surrogate endpoint / similar intervention) → "Conditional"
  - Only expert consensus / guidelines, no direct study evidence → "Consensus-based"
- **Important**: Population mismatch, limited sample size, short follow-up, indirectness — document these in `caveats`. They affect evidence quality, not recommendation strength on their own. Only downgrade strength when benefits vs. harms are genuinely uncertain or results conflict.
- Do NOT upgrade strength based on clinical intuition or training knowledge.

**Logical Consistency Check (MANDATORY):**
- **没有数据 ≠ 没有差异。** If the included evidence did NOT measure a specific outcome, you MUST NOT claim "无显著差异" for that outcome. Correct: "现有证据未报告该结局的直接比较数据". Only say "无显著差异" when evidence actually measured the outcome AND showed p>0.05 or CI crossing the null.
- **recommendation 与 caveats 不得矛盾。** If caveats list "无[X]直接数据", the recommendation text MUST NOT claim "[X]无显著差异". If caveats note population mismatch, recommendation must note limited generalizability.
- **SMD > 3 须标注。** Report as "该数值异常偏大，可能为原始均差（mmHg）而非标准化均差，效应量解释受限".

**Step 4 - Determine Adopted Evidence Quality:**
Based on Steps 1-3, identify which evidence items you are actually adopting for this recommendation (i.e., evidence that is sufficiently direct and relevant to the PICO). Report the collective GRADE quality of ONLY the adopted evidence as `evidence_quality`. Do NOT average in evidence that you rejected due to population mismatch, wrong comparator, or irrelevant outcomes.
- If strength is "Insufficient Evidence" or "No Recommendation" → set `evidence_quality` to "Very Low"
- If only indirect evidence was adopted (Conditional) → set `evidence_quality` to the grade of that indirect evidence

**Step 5 - Address Previous Feedback:**
If there is "Previous Attempt Feedback", explicitly acknowledge each issue and explain how you are correcting it.

**Step 6 - Evidence Gap Search Strategy (for NOT_COVERED outcomes only):**
For each outcome with `status: "NOT_COVERED"` in your `outcome_coverage`:
- Generate a PubMed search strategy using MeSH terms and Boolean operators
- Include relevant filters (e.g., study type, date range)
- Output as `gap_searches` in the JSON

## 【可读性模板 — recommendation 字段必须严格遵循】

recommendation 字段是面向临床医生的最终输出，必须高度可读。请严格按以下模板生成：

**格式规则：**
1. 使用【段落标题】标记每个段落，段落之间用换行符 `\n\n` 分隔
2. 段落内部使用 `\n• ` 开头的项目符号列出要点（注意：`\n` + `• ` + 内容）
3. 关键数据用括号标注：(HR 0.99, 95%CI 0.91-1.08)
4. 引用使用短格式：[作者 年份]（如 [SALIM 2008]）
5. 每个要点控制在一行以内（<80字符）
6. 若某段落无数据则省略该段落

**模板结构：**

```
【结论】\n一句话回答问题。推荐强度：X，证据质量：Y。\n\n【疗效】\n• 要点1：关键数据 (效应量, 95%CI) [短引用]\n• 要点2：...\n\n【安全性】\n• 不良反应1：发生率对比 [短引用]\n• 不良反应2：...\n\n【临床建议】\n• 适用人群：...\n• 监测要点：...\n• 禁忌/注意：...
```

**示例输出（recommendation字段的值）：**

```
【结论】\nARB与ACEI均可作为无合并症高血压患者的一线选择。推荐强度：Conditional，证据质量：Moderate。\n\n【疗效】\n• 心血管复合终点：两药无显著差异 (HR 0.99, 95%CI 0.91-1.08) [SALIM 2008]\n• 卒中预防：ARB可能更优 (HR 0.68, 95%CI 0.52-0.89)，但非头对头比较 [POPAT 2025]\n• 血压达标率：无直接对比数据\n\n【安全性】\n• 干咳致停药：ACEI显著高于ARB (12.3% vs 4.7%) [SALIM 2008]\n• 高钾血症：无直接对比数据\n\n【临床建议】\n• 适用人群：无肾动脉狭窄、血管性水肿史、妊娠的成人高血压患者\n• 选药倾向：咳嗽不耐受→ARB；CKD+蛋白尿→优先ARB\n• 监测：起始1-2周查肌酐/eGFR/血钾\n• 禁忌：妊娠/备孕禁用，双侧肾动脉狭窄慎用
```

**常见错误（禁止）：**
- ❌ 把所有内容写成一个长段落
- ❌ 在 recommendation 中使用完整 evidence_id（如 [EV-RCT-2008-SALIM-001 / results_1]）
- ❌ 重复 rationale 中的详细论证过程
- ❌ 把 caveats 内容混入 recommendation 文本
- ❌ 写超过6个要点的段落（信息过载）
- ❌ 对无数据的结局说"无显著差异"（应说"无直接对比数据"）
